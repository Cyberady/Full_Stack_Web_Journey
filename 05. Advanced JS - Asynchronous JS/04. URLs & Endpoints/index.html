URLs & API Endpoints

Learn how applications communicate with servers using URLs and API endpoints.

What is a URL?

URL stands for Uniform Resource Locator.

A URL is the address used to access resources on the internet. 

https://example.com/users/profile

Parts of a URL

  • https:// → Protocol
  • example.com → Domain Name
  • /users/profile → Path

What is an Endpoint?

An endpoint is a specific URL where an API can access a resource.

APIs expose multiple endpoints for different operations like getting users, posts, products, or authentication data. 

https://api.example.com/users

Endpoint Examples

GET /users

Fetch all users

POST /users

Create a new user

PUT /users/1

Update user data

DELETE /users/1

Delete a user

How It Works

Client

Browser or App sends request

➡

Endpoint

Server receives request at endpoint

➡

Response

Server sends back JSON or data

URLs help locate resources, while endpoints help APIs provide data.
